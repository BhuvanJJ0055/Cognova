# Medical Q&A Retrieval Advisor (Task 2)

**Author**: Bhuvan J J  
**Workspace**: Cognova  
**Domain**: Artificial Intelligence / Information Retrieval / Natural Language Processing  

This module implements a specialized Medical Question Answering Chatbot using the MedQuAD dataset, featuring a TF-IDF retrieval mechanism, basic medical entity recognition (symptoms, diseases, treatments), and an interactive Streamlit UI.

---

## 1. Problem Statement
Medical search engines and general chatbots frequently hallucinate or return vague answers that lack credibility. This task implements a specialized medical information advisor that:
1. **Provides Credible Answers**: Uses the curated **MedQuAD (Medical Question Answering Dataset)** containing 16,407 validated Q&A pairs.
2. **Retrieves Semantically Matching QA**: Uses TF-IDF vectorization and cosine similarity to quickly match natural language queries against indexed questions.
3. **Tags Medical Entities**: Extracts diseases, symptoms, and treatments using regular expressions with word boundary guards and term-length prioritization.
4. **Delivers an Interactive UI**: Implements a Streamlit dashboard showing color-coded entity badges, match similarity percentages, and a clear medical disclaimer.

---

## 2. Dataset
This project is built on the **MedQuAD** dataset (Ben Abacha et al., 2019), which compiles medical question-answering pairs from 12 trusted NIH websites (e.g., Cancer.gov, Genetic and Rare Diseases Information Center, MedlinePlus).
- **Format**: Originally distributed as a collection of 11,274 XML files.
- **Ingestion Strategy**: We write a data loader that scans the XML files, extracts the `<Focus>` (e.g. disease name), the `<Question qtype="...">` (the query and its intent), and the `<Answer>` text, exporting it to a single flat CSV (`data/medquad_qa.csv`).
- **Lean Distribution**: A curated `sample_medquad_qa.csv` containing 40 representative pairs is included in this repository to make the app work out-of-the-box. The `data_loader.py` script includes a helper to automatically download and extract the full dataset from GitHub on demand.

## 4. Setup & Running Instructions

### Install Dependencies
Activate your virtual environment and install the required libraries:
```bash
pip install -r requirements.txt
```

### Run Automated Tests
Verify that the entity tagging and semantic search indexes work correctly:
```bash
python test_entities.py
python test_pipeline.py
```

### Launch the Streamlit App (Standalone)
Run the dedicated medical interface:
```bash
streamlit run app.py
```

### Ingesting the Full Dataset
By default, the application runs on the compiled sample subset of 40 entries. To parse and run queries against the full 16,407 Q&A pairs:
1. Open the sidebar in the Streamlit app.
2. Click **"Download & Build Full Index"**.
3. The script will fetch the MedQuAD master repository from GitHub, extract the XML files, parse them, export them to a 24MB CSV, and build a 26MB TF-IDF index file in under a minute.

## 5. Methodology & Implementation

### A. Medical Entity Recognition (`entity_recognition.py`)
Matches keywords inside the cleaned query. To prevent false positives:
- Uses regular expressions with word boundary markers `\b` (e.g. preventing the word "pain" from matching "painting" or "inhaler" from matching "inhalation").
- Disease list is dynamically generated by extracting all unique `<Focus>` values from the dataset, matching actual medical terms (e.g., "celiac disease", "type 2 diabetes").
- Longest multi-word terms are matched first to ensure specific compound entries are tagged instead of their smaller constituent words.

### B. Retrieval Mechanism (`build_index.py`)
- Employs `scikit-learn`'s `TfidfVectorizer` (configured with `ngram_range=(1,2)` to capture negations and stop words filtered out).
- Computes cosine similarity between the query vector and the vector matrix.
- Sorts descending and applies a configurable minimum similarity cutoff threshold (e.g., 0.15).
